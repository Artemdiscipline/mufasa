# /assets — что сюда положить

Сейчас на странице все фото — серые плейсхолдеры (рисуются через CSS, файлов не требуют).
Рядом с каждым в `index.html` стоит комментарий `<!-- TODO: заменить ... -->`.
Чтобы подставить реальные материалы, положите файлы по этим путям — код их подхватит.

## Логотип  ← нужен один файл: `assets/logo.png`
Присланный логотип — растровый (чёрная эмблема на белом фоне), поэтому:
- сохраните его как **`assets/logo.png`** (именно png, именно сюда);
- он уже подключён в герое и прелоадере;
- поскольку он чёрный на белом, в CSS стоит `filter: invert(1)` — на тёмном фоне он
  показывается белым, а белый фон сливается с фоном страницы;
- если появится версия **с прозрачным фоном** (или белый лев на прозрачном) — положите её
  тем же именем и уберите `filter:invert(1)` в двух местах (`.preloader__logo` и `.hero__logo`).

По брифу для растрового логотипа «сборка гривы из клиньев» в прелоадере отключена —
вместо неё логотип раскрывается маской снизу вверх + счётчик 00→100.
Если пришлёте **`logo.svg`** (векторную эмблему) — сможем вернуть анимацию сборки гривы.

## Изображения — `assets/img/`
Все фото остаются ЦВЕТНЫМИ (интерфейс монохромный, фотографии — нет). Формат WebP.

| Файл | Назначение | Пропорции / размер |
|------|------------|--------------------|
| `lion.webp`   | Портрет/эмблема льва в герое | ~760×980, вертикальный |
| `father.webp` | «Отец и сын» — взрослый в кресле | 3:4, ~360×480 |
| `son.webp`    | «Отец и сын» — мальчик в кресле | 3:4, ~360×480 |
| `dinar.webp`  | Мастер Динар | 3:4 |
| `radik.webp`  | Мастер Радик | 3:4 |
| `master-3.webp`, `master-4.webp` | Остальные мастера (имена уточнить) | 3:4 |
| `hall-1..5.webp` | Секция «Атмосфера»: зал, сосновая доска, мох, кресло, свет | разные |
| `map.webp`    | Статичный скриншот карты (обрабатывается в grayscale в CSS) | 4:3 |
| `og.jpg`      | Open Graph превью (белый лев на чёрном) | 1200×630 |

После добавления файлов замените соответствующий блок `<div class="ph">…</div>`
на `<img src="assets/img/ИМЯ.webp" alt="…" width="…" height="…" loading="lazy">`
(у героя — `fetchpriority="high"` вместо `loading="lazy"`).
Обёртки уже имеют нужные маски и reveal-анимации — менять их не нужно.
